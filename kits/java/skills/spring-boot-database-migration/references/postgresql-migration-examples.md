# PostgreSQL Changeset Examples

Runnable SQL changesets targeting PostgreSQL. Files are named `NNNN__description.sql` (4-digit monotonic sequence) and executed manually by the DBA — see the parent SKILL.md for the workflow.

## 0001__init_schema.sql — Initial Baseline

Baseline schema with users, roles, and user_role junction table:

```sql
-- 0001__init_schema.sql
-- Ticket: PROJ-001 — initial baseline

CREATE TABLE IF NOT EXISTS users (
    id          BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    username    TEXT    NOT NULL UNIQUE,
    email       TEXT    NOT NULL UNIQUE,
    password    TEXT    NOT NULL,
    status      TEXT    NOT NULL DEFAULT 'ACTIVE'
                CHECK (status IN ('ACTIVE', 'SUSPENDED', 'DELETED')),
    created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
    created_by  BIGINT  NOT NULL,
    updated_by  BIGINT  NOT NULL,
    deleted_at  TIMESTAMPTZ,
    version     INTEGER NOT NULL DEFAULT 1
);

CREATE TABLE IF NOT EXISTS roles (
    id          BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    name        TEXT    NOT NULL UNIQUE,
    description TEXT,
    created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE IF NOT EXISTS user_role (
    user_id     BIGINT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    role_id     BIGINT NOT NULL REFERENCES roles(id) ON DELETE CASCADE,
    PRIMARY KEY (user_id, role_id)
);

CREATE INDEX IF NOT EXISTS idx_users_email       ON users (email);
CREATE INDEX IF NOT EXISTS idx_users_deleted_at  ON users (deleted_at) WHERE deleted_at IS NULL;
CREATE INDEX IF NOT EXISTS idx_users_status      ON users (status);
CREATE INDEX IF NOT EXISTS idx_user_role_role_id ON user_role (role_id);
```

## 0002__add_user_status.sql — ALTER TABLE with Guard

Add columns for user activity tracking:

```sql
-- 0002__add_user_status.sql
-- Ticket: PROJ-118 — track last login & login count

BEGIN;
ALTER TABLE users ADD COLUMN IF NOT EXISTS last_login_time TIMESTAMPTZ;
ALTER TABLE users ADD COLUMN IF NOT EXISTS login_count     INTEGER NOT NULL DEFAULT 0;
COMMIT;

CREATE INDEX IF NOT EXISTS idx_users_last_login ON users (last_login_time DESC NULLS LAST);
```

> `IF NOT EXISTS` on `ALTER TABLE ADD COLUMN` requires PostgreSQL 9.6+. Project targets PG 18+, so always supported.

## 0003__create_index_concurrently.sql — Non-Blocking Index

`CREATE INDEX CONCURRENTLY` cannot run inside a transaction — keep it in a dedicated file with no explicit `BEGIN`:

```sql
-- 0003__create_index_concurrently.sql
-- Ticket: PROJ-205 — case-insensitive username lookup
-- NOTE: CONCURRENTLY — must run outside any transaction block

CREATE INDEX CONCURRENTLY IF NOT EXISTS idx_users_username_lower
    ON users (LOWER(username));
```

DBA executes with `psql` directly (no `-c "BEGIN; ... COMMIT;"` wrapper).

## 0004__add_constraints.sql — CHECK + UNIQUE Constraints

```sql
-- 0004__add_constraints.sql
-- Ticket: PROJ-210 — enforce email format

BEGIN;
ALTER TABLE users
    ADD CONSTRAINT chk_users_email_format
    CHECK (email ~ '^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}$');
COMMIT;
```

## 0005__seed_reference_data.sql — Idempotent INSERT

Seed default roles and configuration with `ON CONFLICT DO NOTHING` so a re-run is safe:

```sql
-- 0005__seed_reference_data.sql
-- Ticket: PROJ-301 — default role catalog & system config

INSERT INTO roles (name, description) VALUES
    ('ADMIN', 'System administrator with full access'),
    ('USER',  'Standard application user'),
    ('GUEST', 'Read-only guest access')
ON CONFLICT (name) DO NOTHING;

CREATE TABLE IF NOT EXISTS system_config (
    id          BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    key         TEXT NOT NULL UNIQUE,
    value       TEXT NOT NULL,
    created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);

INSERT INTO system_config (key, value) VALUES
    ('max_login_attempts',      '5'),
    ('session_timeout_minutes', '30'),
    ('password_min_length',     '8')
ON CONFLICT (key) DO NOTHING;
```

## 0006__alter_column_type.sql — Safe Type Change

```sql
-- 0006__alter_column_type.sql
-- Ticket: PROJ-405 — widen login_count, allow more status values

BEGIN;

-- Widening is safe — BIGINT accepts all INTEGER values
ALTER TABLE users ALTER COLUMN login_count TYPE BIGINT;

-- Widen CHECK constraint to allow additional states
ALTER TABLE users DROP CONSTRAINT IF EXISTS chk_users_status_values;
ALTER TABLE users ADD  CONSTRAINT chk_users_status_values
    CHECK (status IN ('ACTIVE', 'SUSPENDED', 'DELETED', 'PENDING', 'LOCKED'));

COMMIT;
```

> **Destructive type changes** (TEXT → narrower, incompatible) require a multi-step runbook executed by DBA — not a single changeset. Approach: (1) add a new column, (2) backfill in batches outside changesets, (3) verify, (4) drop old column in a later release. Each step is its own file.

## Additional Patterns

### Many-to-many junction table

```sql
CREATE TABLE IF NOT EXISTS user_permission (
    user_id       BIGINT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    permission_id BIGINT NOT NULL REFERENCES permissions(id) ON DELETE CASCADE,
    granted_by    BIGINT NOT NULL,
    granted_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
    PRIMARY KEY (user_id, permission_id)
);
CREATE INDEX IF NOT EXISTS idx_user_permission_permission ON user_permission (permission_id);
```

### Partial index for active rows

```sql
CREATE INDEX IF NOT EXISTS idx_users_active_email
    ON users (email) WHERE deleted_at IS NULL;
```

### Generated column (PG 12+)

```sql
ALTER TABLE users ADD COLUMN IF NOT EXISTS email_domain TEXT
    GENERATED ALWAYS AS (SPLIT_PART(email, '@', 2)) STORED;
CREATE INDEX IF NOT EXISTS idx_users_email_domain ON users (email_domain);
```

### Partitioned table for time-series data

```sql
CREATE TABLE IF NOT EXISTS audit_log (
    id          BIGINT GENERATED BY DEFAULT AS IDENTITY,
    action      TEXT    NOT NULL,
    table_name  TEXT    NOT NULL,
    row_id      BIGINT,
    old_data    JSONB,
    new_data    JSONB,
    created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
    created_by  BIGINT  NOT NULL
) PARTITION BY RANGE (created_at);

CREATE TABLE IF NOT EXISTS audit_log_2026_05 PARTITION OF audit_log
    FOR VALUES FROM ('2026-05-01') TO ('2026-06-01');
CREATE TABLE IF NOT EXISTS audit_log_2026_06 PARTITION OF audit_log
    FOR VALUES FROM ('2026-06-01') TO ('2026-07-01');

CREATE TABLE IF NOT EXISTS audit_log_default PARTITION OF audit_log DEFAULT;
```
