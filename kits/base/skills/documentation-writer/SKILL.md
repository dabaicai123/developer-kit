# Documentation Writer (Diátaxis)

Create high-quality software documentation.

## When to use this skill

- Writing tutorials, how-to guides, reference docs, or explanations following the Diátaxis framework
- Creating or updating project documentation, API references, or onboarding guides
- Structuring documentation by audience and purpose (learning vs. problem-solving vs. information vs. understanding)

## Four Document Types

| Type | Purpose | Orientation |
|------|---------|-------------|
| Tutorial | Learning | Practical steps |
| How-to Guide | Problem-solving | Specific problem |
| Reference | Information | Technical descriptions |
| Explanation | Understanding | Clarification |

## Workflow

### 1. Determine Document Type

- **Tutorial**: Learning-oriented, guides newcomers
- **How-to**: Problem-oriented, solves specific issue
- **Reference**: Information-oriented, technical details
- **Explanation**: Understanding-oriented, clarifies concepts

### 2. Identify Audience

- Novice developers
- Experienced users
- Non-technical users

### 3. Define Scope

- What to include
- What to exclude
- User's goal

### 4. Generate Content

Follow guiding principles:
- **Clarity**: Simple, unambiguous language
- **Accuracy**: Correct, up-to-date information
- **User-Centricity**: Focus on user's goal
- **Consistency**: Tone, terminology, style

## Document Patterns

### Tutorial
- Step-by-step practical lessons
- No prior knowledge assumed
- Learning by doing

### How-to Guide
- Solves specific problem
- Assumes some knowledge
- Step-by-step recipe

### Reference
- Comprehensive and complete
- Structured for lookup
- Technical descriptions

### Explanation
- Background and context
- Why and how things work
- Clarifies concepts
